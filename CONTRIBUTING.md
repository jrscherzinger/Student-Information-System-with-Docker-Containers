# Contributing Guide

This document describes how contributors should work on this project.

---

## Branch Strategy

main: stable release branch  
develop: active development branch  
feature/*: new features, fixes, or documentation  

---

## Contribution Rules

All changes must be done in feature branches  
Direct commits to develop and main are not allowed  
Pull Requests are required to merge into develop  
Code review is mandatory before merging  

---

## Commit Rules

Use clear and descriptive commit messages  
Follow a consistent commit style  
Each student is expected to contribute commits weekly  

---

## Pull Request Process

1. Create a feature branch
2. Make and commit your changes
3. Push the branch to GitHub
4. Open a Pull Request targeting develop
5. Merge after review approval
